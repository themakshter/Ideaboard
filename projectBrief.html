Idea Board

Introduction

Many people use boards to stick their notes on. This helps them visualise their train of thought and also helps them keep their ideas organised. An example of such a board is shown below:


We will create a prototype of a note-taking application which will model after such boards. Users will be able to add ideaboards and then add notes to them. These notes will be divided into different columns which can be named by users. The notes will also have a reasonable degree of customisation such as basic text formatting and custom note colours.

User Stories

  1. As a user I want to create an ideaboard to store and organise notes, so that I can get an overview with just one glance.
  2. As a user, I want to be able to make notes in rich text such as headings and bullet points, so that I can customise my notes as I like .
  3. As a user I want to have access to my boards from anywhere, so that I can always check my notes.
  4. As a non-administrative user, I want to view and modify my ideaboards but not the boards of others.
  5. As a developer, I want to make sure that the application works properly on the latest versions of all browsers, so that users can use their preferred browsers.

Technologies

For the back end of our application, we will be using a combination of Backbone.js and node.js while using a MongoDB or SQL database. The front end will be written in a combination of HTML5, CSS3 and JavaScript. Some libraries which will be in use include jQuery,jQuery-Notebook and the Bootstrap framework.

Project Plan

For the coming week or so, we will spend time designing the front end and back end of our application, creating wireframes and going over tutorials and documentation. After that, we will be setting up our server and backend, writing tests for our work in parallel. With our server running, we will build the application itself using HTML, CSS and JavaScript. As our application is mostly visual, this will take most of our time. We will also be aiming to test our application as we go.
As we go through our project, we will aim to write documentation in parallel so that we do not have to rush at the end. Therefore the technical manual will start the moment we initiate setting up our back end and we will start writing the user manual once roughly halfway through our front end application design.
